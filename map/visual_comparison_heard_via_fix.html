How the Fix Works

  1. Track nodes with neighbors: During neighbor link creation, add each node to a Set called nodesWithNeighbors
  2. Skip these nodes: When creating "heard via" links, check if the node is in nodesWithNeighbors
  3. Only infer for unknown: Create brown "heard via" links ONLY for nodes NOT in the Set (nodes D and E)
  4. Result: Nodes with actual neighbor data (B and C) don't get redundant inferred links
